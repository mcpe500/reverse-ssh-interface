Below is a **complete, practical monorepo file structure** for your project that matches your folders (`cli/ core/ gui/ web/ docs/`) and supports:

* **Shared core logic** in Rust (`core/`)
* **CLI** (Rust) (`cli/`)
* **Web server** (Rust `axum`) + **Web UI** (SvelteKit or React) (`web/`)
* **GUI** (Tauri) reusing the same Web UI build (`gui/`)
* Packaging, CI, release, and sane defaults

I’m assuming **Rust workspace** at the root.

## Notes on a few key structure decisions

* **`core/` is a library crate**. Everything important lives here: ssh detection, args building, spawning, supervision, events.
* **`cli/` is thin**: parse args, call core, render output.
* **`web/server/` is thin**: exposes HTTP/WS to control core + serves static UI.
* **`web/ui/` is your actual web app** (React/Svelte). It talks to localhost backend.
* **`gui/` uses Tauri**; you can reuse the same UI build output from `web/ui` to avoid duplicating frontend work.
